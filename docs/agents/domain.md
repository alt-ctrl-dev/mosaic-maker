# Domain Docs

This repository uses a single-context domain documentation layout.

## Before exploring

Read these when they exist:

- `CONTEXT.md` at the repository root
- Relevant ADRs under `docs/adr/`

If they do not exist, proceed silently. Domain-modeling skills create them only when terminology or decisions need recording.

## Use the glossary vocabulary

Use terms as defined in `CONTEXT.md` across code, tests, issues, and specifications. Avoid synonyms that the glossary explicitly rejects.

## Flag ADR conflicts

Explicitly identify proposals that contradict an existing ADR rather than silently overriding the decision.
